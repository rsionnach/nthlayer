# NthLayer v1.5 Epic Tree

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the three-tier NthLayer runtime (core, workers, bench) with v1.5 capabilities — string-ID verdicts, JSON encoding, asyncio correlation, CloudEvents envelopes, unified store, Instructor-backed LLM calls — leaving IPLD CIDs, CBOR, Rekor, Biscuit/Regorus for v2.

**Architecture:** Three-tier model. Tier 1 (nthlayer-core) owns the unified SQLite store, HTTP API, retention, and heartbeat monitoring. Tier 2 (nthlayer-workers) runs observe/measure/correlate/respond/learn modules that communicate with core exclusively via HTTP API. Tier 3 (nthlayer-bench) is a Textual TUI that communicates with core via HTTP API. Seven repos: opensrm, nthlayer-common, nthlayer-generate, nthlayer (core), nthlayer-workers, nthlayer-bench, nthlayer-site.

**Tech Stack:** Python 3.11+, SQLite WAL, Starlette (core API), httpx, Instructor, Textual, prometheus-api-client, scipy, networkx, opentelemetry-sdk.

**Key Specs:**
- Runtime architecture: NTHLAYER-SERVE-MODE-v2.1
- Shared library: NTHLAYER-COMMON-v1
- Verdict model: NTHLAYER-LEARN-v1
- Measurement: NTHLAYER-MEASURE-v1
- Correlation: NTHLAYER-CORRELATE-v1
- Operator UI: NTHLAYER-BENCH-v2.1
- Wire format: NTHLAYER-TELEMETRY-ENVELOPE-v1
- Manifest spec: OPENSRM-CORE-v2
- Auth spec: OPENSRM-RBAC-EXTENSION-v2

#### P1-B.3: Core HTTP API — verdict and assessment endpoints

- **Spec:** NTHLAYER-SERVE-MODE-v2.1 §3, NTHLAYER-LEARN-v1 §4.5 (outcome resolution), spec revision summary §3 (workers use API)
- **Task:** In `nthlayer/src/nthlayer/api.py`, implement Starlette routes:

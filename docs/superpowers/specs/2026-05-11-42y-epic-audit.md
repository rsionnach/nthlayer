# opensrm-42y Epic Audit — 2026-05-11

## Context

The **opensrm-42y "Demo Improvement: Accountability & Portfolio Story"**
epic was authored 2026-04-16, ten days before the workers consolidation
landed on 2026-04-26 (opensrm-hty.7 / RM.7). Five repos
(`nthlayer-observe`, `nthlayer-measure`, `nthlayer-correlate`,
`nthlayer-respond`, `nthlayer-learn`) merged into `nthlayer-workers`;
the standalone `nthlayer-observe` console script was retired. The
epic's driving spec, `DEMO-IMPROVEMENT-SPEC.md`, was moved to
`nthlayer/docs/archived-specs/` and never refreshed against the new
architecture.

A second sub-track inside the same epic (`42y.10–.15`) references a
React Flow migration spec (`REACT-FLOW-MIGRATION-SPEC.md`) that does
not exist anywhere in the workspace and a target directory
(`nthlayer-site/demo/`) that lives in a separate repo not present in
this clone.

This audit walks every open child and produces a verdict so the epic
can be unblocked or formally archived rather than left in limbo.

## Recommended actions

1. **Adopt one architectural pattern across .3/.4/.6.** The choice is
   not purely architectural — it also drives what the demo *says* about
   NthLayer:
   - **(a) on-demand CLI invocations** narrates *"operator has tools"*
     — the demo shows a human probing the system at each beat.
   - **(b) poll worker-emitted assessments** narrates *"system is
     always observing"* — the demo shows a continuous-observation
     runtime that an operator can read from at any beat.

   **Recommend (b) — poll worker-emitted `portfolio_status`
   assessments.** Three reasons:

   1. `demo.sh` already polls core for `portfolio_status`, so (b) is
      the lower-friction path.
   2. The continuous-observation pattern is more architecturally
      distinctive than ad-hoc CLI invocation — it demonstrates the
      worker-module pattern central to NthLayer's value proposition.
   3. (a) would require new CLI invocation infrastructure
      post-consolidation (the `nthlayer-observe` console script was
      retired; only `python -m nthlayer_workers.observe.cli` works).

   **Trade-off accepted:** the demo narrative shifts from "operator
   probes system" to "system continuously emits state observable by
   operator." The continuous-observation framing is stronger for v1.5
   positioning.

2. **Refresh the spec, or de-spec.** `DEMO-IMPROVEMENT-SPEC.md` is
   archived. Either move the still-valid narrative beats into a new
   spec under `docs/roadmap/` (or directly into the beads
   themselves), or accept that the beads are their own spec.

3. **Close the React Flow sub-track (.10–.15) as
   design-exploration.** Reasoning:
   - `nthlayer-site` is a separate concern per the front-door's
     `CLAUDE.md`.
   - `REACT-FLOW-MIGRATION-SPEC.md` does not exist, so the work
     isn't concretely scoped.
   - Moving placeholder beads into a new tracker would move
     placeholder commitments, not real work.
   - Migration to `nthlayer-site`'s tracker happens later — when
     the spec exists and the work is scoped.

   Closing as design-exploration acknowledges the ideas existed
   without pretending they're actionable. Mixing them with the CLI
   sub-track creates false dependencies (.13 cites both specs).

4. **Close .2 as obsolete** with a note pointing to this audit. The
   bead's stated intent has been satisfied by a different mechanism.

5. **Rewrite .1, .3, .4, .6** to match the chosen architectural
   pattern.

6. **Verify .5 in place** — quick check that `nthlayer-workers gate`
   returns exit 2 and surfaces a hash, then close.

7. **Defer .7 and .9** until the rewrites land.
